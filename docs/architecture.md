# 스위치온 다이어트 앱 아키텍처 (Architecture)

## 1. 개요
본 애플리케이션은 박용우 원장의 '스위치온 다이어트' 4주 프로그램을 사용자가 쉽게 따라하고 기록할 수 있도록 돕는 모바일 웹 애플리케이션입니다. React와 Tailwind CSS를 기반으로 구축되었으며, 상태 관리는 React Hooks를 활용합니다.

## 2. 주요 기능 및 화면 구성
앱은 하단 네비게이션 바(BottomNav)를 통해 4개의 메인 탭으로 구성됩니다.

### 2.1. 홈 화면 (HomeScreen)
- **목적:** 오늘의 다이어트 현황 파악 및 데일리 미션 수행
- **주요 컴포넌트:**
  - **상단 요약 카드:** 현재 주차/일차, 남은 공복 시간 타이머 표시
  - **데일리 미션 체크리스트:** 물 2L, 영양제, 고강도 운동, 수면(자동 계산) 달성 여부 체크
  - **식단 타임라인:** 아침, 점심, 간식, 저녁 식사 스케줄 및 기록 상태 표시
  - **수면/기상 관리:** 취침 시간 경고, 기상 시간 입력 및 수면 시간 자동 계산 로직 포함

### 2.2. 캘린더 화면 (CalendarScreen)
- **목적:** 28일 챌린지의 전체 진행 상황 및 일별 상세 기록 확인
- **주요 컴포넌트:**
  - **월간 달력 뷰:** 일별 성공/주의/미입력 상태를 색상과 아이콘으로 직관적 표시
  - **일별 상세 뷰:** 선택한 날짜의 식단 기록 및 데일리 규칙(물, 영양제, 운동, 수면) 달성도 표시
  - **AI 코치 피드백:** 해당 일자의 종합 평가 (완벽 성공/보통/최악) 및 놓친 규칙 안내
  - **주차별 현황 프로그레스 바:** 1~4주차 진행률 표시

### 2.3. 식단 기록 화면 (DietScreen / RecordMealScreen)
- **목적:** 식사 사진 촬영 및 상세 내용 기록
- **주요 컴포넌트:**
  - **카메라/갤러리 연동:** (현재 UI 목업으로 구현)
  - **식단 텍스트 입력:** 먹은 음식 텍스트 기록
  - **AI 분석 피드백:** 기록된 식단이 해당 주차의 허용 식품인지 분석 (목업)

### 2.4. 마이페이지 (MyScreen)
- **목적:** 사용자 설정 및 전체 기록 모아보기
- **주요 컴포넌트:**
  - **기본 수칙 요약:** 스위치온 다이어트 7대 수칙 리마인드
  - **식단 및 수면 기록 보기 (DietHistoryScreen):** 과거의 모든 식단, 수면, 데일리 미션 달성 여부를 리스트 형태로 확인
  - **시작일 설정 및 데이터 초기화:** 다이어트 시작일 변경 및 전체 데이터 리셋 기능 (SetupScreen 연동)

## 3. 데이터 모델 (Data Model)
현재는 프론트엔드 상태(State)와 Mock Data로 관리되고 있으며, 추후 백엔드 연동 시 아래 구조를 따릅니다.

- **User:** 시작일, 기상/취침 목표 시간
- **DailyRecord:**
  - `date`: 날짜
  - `wakeTime`, `bedTime`: 기상/취침 시간
  - `rules`: `{ water: boolean, supplements: boolean, workout: boolean, sleep: { h, m, valid } }`
  - `meals`: `Array<{ type, time, status, text }>`
  - `feedback`: `{ status: 'perfect' | 'normal' | 'worst', text: string, missed: string[] }`

## 4. 기술 스택
- **Frontend:** React 18, Vite, Tailwind CSS, Lucide React (Icons)
- **Routing:** 상태 기반 조건부 렌더링 (`activeTab`, `activeScreen` in `App.tsx`)
