# 🔧 스위치온 다이어트 앱 - 개발자 모드 가이드라인

본 문서는 스위치온 다이어트 앱의 **개발자 모드(Developer Mode)**에 대한 상세 가이드라인입니다.
앱의 QA(품질 보증), 디버깅, 그리고 다양한 엣지 케이스(Edge Case) 테스트를 위해 설계되었습니다.

---

## 1. 개발자 모드 개요
개발자 모드는 실제 사용자가 겪을 수 있는 다양한 상황(식사 누락, 비정상적인 수면 시간, 과거 데이터 조회 등)을 강제로 시뮬레이션하고, 데이터베이스(DB)에 저장되는 Raw 데이터를 직접 확인 및 조작하기 위해 만들어졌습니다.

*   **접근 방법:** 앱 하단 네비게이션 `[My]` 탭 클릭 ➔ 마이페이지 맨 하단 `[🔧 개발자 모드]` 버튼 클릭

---

## 2. 주요 기능 상세 가이드

### 2.1. 데이터 조회 날짜 (Day 선택)
현재 앱에서 진행 중인 Day를 기준으로 데이터를 조회합니다.
*   **기본 동작:** 현재 진행 중인 Day(예: Day 1)까지만 바텀 시트(Bottom Sheet) 리스트에 노출됩니다.
*   **미래 날짜 잠금 해제 (Unlock):** 우측의 `[자물쇠 아이콘]`을 클릭하면 Day 1부터 Day 28까지 모든 날짜가 해제됩니다. 이를 통해 미래의 날짜에 미리 데이터를 세팅해보고 UI가 어떻게 반응하는지 테스트할 수 있습니다.

### 2.2. 기상 및 취침 시간 수정
사용자의 수면 데이터를 강제로 조작하여 AI 코치의 피드백 변화를 테스트합니다.
*   **조작 방법:** 각 항목 우측의 `[Edit(연필) 아이콘]` 클릭
*   **입력 방식:** 모바일 네이티브 Time Picker(시/분 선택기)를 사용하여 직관적으로 시간을 변경할 수 있습니다. (초 단위는 `00`으로 자동 고정됩니다.)

### 2.3. 식사 기록 강제 조작
식사 상태(대기, 완료, 스킵)를 자유롭게 변경하여 식단 진행률 및 UI 상태를 테스트합니다.
*   **[Record] (보라색):** 해당 식사를 '기록 완료' 상태로 변경하며, 시간은 **현재 시간**으로 자동 입력됩니다.
*   **[Skip] (회색):** 해당 식사를 '스킵' 상태로 변경하며, 사유는 `강제 스킵 (개발자)`로 고정 입력됩니다.
*   **[시간 수정] (연필 아이콘):** 이미 기록된 식사(`Record`)의 경우, 연필 아이콘을 눌러 기록된 시간을 과거/미래로 덮어쓸 수 있습니다.
*   **[기록 삭제] (휴지통 아이콘):** 기록되거나 스킵된 식사를 다시 초기 상태(`아직 안 먹음`)로 되돌립니다.

### 2.4. 체중 및 체지방 수정
*   **조작 방법:** `[Edit(연필) 아이콘]` 클릭 후 숫자(소수점 포함) 입력.
*   **목적:** 체중 변화 그래프나 목표 달성률 UI를 테스트하기 위해 사용합니다.

---

## 3. Raw JSON (DB 구조 확인)
현재 화면에서 조작한 데이터가 실제 데이터베이스(DB)에 어떤 형태(Schema)로 저장되는지 실시간으로 확인하는 영역입니다.

*   **읽기 전용 (Read-only):** 이 영역은 직접 텍스트를 타이핑해서 수정하는 곳이 아닙니다. 위의 UI 버튼들로 조작한 결과가 DB 포맷(ISO 8601 DateTime 등)으로 어떻게 변환되는지 **눈으로 확인**하는 용도입니다.
*   **[Copy] 버튼:** 현재 JSON 데이터를 클립보드에 복사합니다. 에러 리포팅 시 유용합니다.
*   **[Export] 버튼:** `diet_record_dayX.json` 파일 형태로 기기에 다운로드합니다. 백업 및 외부 툴 분석용으로 사용됩니다.

---

## 4. 글로벌 액션 (데이터 영구 적용 및 삭제)

화면 하단에 위치한 두 개의 큰 버튼은 **실제 DB에 직접적인 영향**을 미칩니다.

*   **[변경사항 DB에 강제 저장] (보라색 버튼):**
    *   화면에서 수정한 모든 임시 데이터(기상, 취침, 식사, 체중 등)를 실제 DB에 덮어씁니다.
    *   이 버튼을 누르기 전까지는 화면에만 임시로 반영된 상태입니다.
*   **[Day 데이터 완전 삭제] (빨간색 버튼):**
    *   현재 선택된 Day의 모든 데이터를 초기화합니다. (수면 시간 `--:--`, 모든 식사 `pending` 상태)
    *   실수로 누르는 것을 방지하기 위해 경고 다이얼로그가 먼저 노출됩니다.

---

## 5. 테스트 권장 시나리오 (QA Checklist)
1.  **수면 부족 시나리오:** 취침 시간을 늦게, 기상 시간을 일찍 설정하여 수면 시간이 6시간 미만일 때 홈 화면의 경고 UI 및 AI 피드백이 정상 노출되는지 확인.
2.  **단식 시간 위반 시나리오:** 전날 마지막 식사 시간과 오늘 첫 식사 시간의 간격을 14시간 미만으로 조작하여 경고가 뜨는지 확인.
3.  **미래 날짜 테스트:** 자물쇠를 풀고 Day 28(마지막 날)로 이동하여 최종 결과 화면이나 축하 UI가 정상 작동하는지 확인.
